# Project Principles

## Engineering over intuition

Career decisions should be based on evidence rather than assumptions.

---

## Build the smallest thing that creates value

Avoid unnecessary complexity.
Every feature should solve a real user problem.

---

## Documentation serves implementation

Documentation exists to improve clarity and execution, not to satisfy process.

---

## AI augments human thinking

The framework uses AI to organize and amplify human knowledge, not replace judgment.

---

## Reusable assets over one-off outputs

The goal is not to generate a single CV.

The goal is to build reusable career assets that support many future applications.

---

## Systems over documents

The framework treats career development as a system rather than a collection of disconnected files.

---

## Continuous evolution

Career assets should evolve together with the professional, not be recreated from scratch every time.

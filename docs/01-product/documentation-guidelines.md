# Documentation Guidelines

## Why this document exists

Consistent documentation improves readability, simplifies collaboration, and allows product decisions to evolve transparently over time.

This document defines the documentation standards used throughout the Career Engineering Framework repository.

## Documentation Principles

Documentation should:

- explain why before describing how;
- evolve together with the product;
- remain concise and easy to navigate;
- reflect current product decisions.

Documentation is treated as part of the product rather than supporting material.
